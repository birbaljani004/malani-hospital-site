# मालाणी हॉस्पिटल — वेबसाइट (Netlify Static + Admin Panel)

यह वर्ज़न **पूरी तरह static HTML वेबसाइट** है (Express या EJS जैसा कुछ नहीं) — इसलिए Netlify पर बिना किसी परेशानी के काम करेगा। हर पेज (`index.html`, `doctors.html`, वगैरह) एक साधारण फाइल है जिसे Netlify सीधे परोसता है। डायनामिक हिस्सा (डॉक्टर, नोटिफिकेशन, फोटो अपलोड, कंटेंट एडिटिंग) सिर्फ छोटे-छोटे Netlify Functions के ज़रिए काम करता है, जो ब्राउज़र से JavaScript के ज़रिए कॉल होते हैं।

## Netlify पर डिप्लॉय कैसे करें

1. इस पूरे फोल्डर को एक **GitHub repository** में डालें।
2. [netlify.com](https://netlify.com) पर "Add new site" → "Import an existing project" → अपनी repo चुनें।
3. `netlify.toml` पहले से तैयार है — कुछ बदलने की ज़रूरत नहीं, बस "Deploy" दबाएं।
4. 1-2 मिनट में साइट लाइव हो जाएगी।
5. **ज़रूरी:** Netlify डैशबोर्ड → Site configuration → Environment variables → नया वेरिएबल जोड़ें:
   - Key: `JWT_SECRET`
   - Value: कोई भी लंबी रैंडम स्ट्रिंग
   
   फिर Deploys → Trigger deploy से दोबारा डिप्लॉय करें।

बस। डेटाबेस/फोटो स्टोरेज अपने आप Netlify Blobs से जुड़ जाता है — कुछ और सेटअप नहीं करना।

## टेस्टिंग यह कैसे की गई

चूंकि सैंडबॉक्स वातावरण में `netlify dev` CLI पूरी तरह शुरू नहीं हो पा रहा था (नेटवर्क पाबंदियों के कारण), हर हिस्से को दूसरे भरोसेमंद तरीकों से टेस्ट किया गया:
- हर Netlify Function को सीधे उसी तरह के इवेंट के साथ चलाकर टेस्ट किया गया जैसे असली Netlify चलाता है (लॉगिन, डॉक्टर जोड़ना/एडिट/डिलीट, फोटो अपलोड सहित)
- एक असली ब्राउज़र-जैसे DOM (jsdom) में असली पेज लोड करके, JavaScript को असल में चलाकर देखा गया कि डॉक्टर कार्ड्स, नोटिस बार, और एडमिन पैनल सही से काम कर रहे हैं
- इसी टेस्टिंग के दौरान 2 असली बग मिले और ठीक किए गए (एक HTML में एक गलत बंद होने वाला टैग, एक JS फाइल में एक missing function declaration, और फोटो के नाम में हिंदी अक्षर होने से आने वाली एक समस्या)

फिर भी, असली Netlify सर्वर पर डिप्लॉय करने के बाद कुछ भी अनपेक्षित दिखे तो बेझिझक बताएं।

## क्या-क्या एडमिन पैनल से किया जा सकता है
- 👨‍⚕️ डॉक्टर जोड़ना/हटाना/एडिट करना (फोटो सहित)
- 📢 नोटिस बार की सूचनाएं जोड़ना/एडिट करना
- 🖼️ साइट की भवन/बैनर फोटो बदलना
- 📝 हर पेज का टेक्स्ट कंटेंट एडिट करना
- 🔒 एडमिन पासवर्ड बदलना

## फोटो अपलोड — साइज़ की चिंता न करें
फोटो चुनते ही वह अपने आप ब्राउज़र में ऑप्टिमाइज़ हो जाती है (resize/compress), चाहे मोबाइल की 15-20MB की फोटो ही क्यों न हो। इससे Netlify की तकनीकी सीमा (लगभग 6MB प्रति रिक्वेस्ट) कभी आड़े नहीं आती।

## सुरक्षा (Security)
- पासवर्ड bcrypt से हैश होकर सेव होता है
- एडमिन लॉगिन token सुरक्षित JWT से बनता है (ब्राउज़र के localStorage में सेव होता है)
- ⚠️ ईमानदारी से: कोई भी वेबसाइट "100% हैक-प्रूफ" नहीं होती। पासवर्ड मज़बूत रखें और `JWT_SECRET` किसी के साथ शेयर न करें।

## डेटा कहां सेव होता है
- डॉक्टर, नोटिफिकेशन, पेज कंटेंट, पासवर्ड → Netlify Blobs (`malani-hospital-db`)
- अपलोड की गई फोटो → Netlify Blobs (`malani-hospital-uploads`)
- मूल थीम फोटो, CSS, JS, शुरुआती डॉक्टर पोस्टर → `public/` फोल्डर में (GitHub repo का हिस्सा)

## संरचना
```
netlify.toml                  → Netlify बिल्ड/रूटिंग सेटिंग
netlify/functions/            → हर एक छोटा, standalone Function (login, doctors CRUD, आदि)
serverdata/store.js                 → Netlify Blobs डेटा लेयर
serverdata/default-db.json          → शुरुआती डेटा
public/                       → पूरी static वेबसाइट (हर पेज असली .html फाइल)
public/admin/                 → एडमिन पैनल (static HTML + JS)
public/assets/site-data.js    → पेजों में डायनामिक डेटा भरने वाली स्क्रिप्ट
```

## अपने कंप्यूटर पर टेस्ट करना
```
npm install -g netlify-cli
npm install
netlify dev
```
`http://localhost:8888` पर पूरी साइट + एडमिन पैनल खुल जाएगी।
